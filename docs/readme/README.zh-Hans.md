# Flang

**让国旗重新回到 macOS 输入源切换器。**

[English](../../README.md) | [Español](README.es.md) | [Français](README.fr.md) | [Português](README.pt-BR.md) | 中文 | [日本語](README.ja.md) | [Русский](README.ru.md)

在 macOS 12.4 之前，菜单栏中的键盘布局切换器会显示一个国家的国旗。之后 Apple
用纯文本标签（"ABC"、"ES"）取代了国旗。Flang 是一个小巧的菜单栏应用，把国旗带
了回来——并让你自己决定用哪面国旗、哪个名称来代表你的每一种语言。

## 功能特性

| | |
|---|---|
| 菜单栏国旗 | 当前输入源以国旗显示，切换时即时更新 |
| 熟悉的切换方式 | 点击即可查看并切换所有输入源，和系统菜单一样 |
| 灵活的指示器 | 国旗与名称可独立显示——任选其一、两者都显示，或都不显示（回退到系统图标） |
| 两种国旗样式 | 扁平国旗图片（经典外观）或原生 emoji 国旗 |
| 合理的默认值 | 每一种 macOS 键盘布局和输入法都会分配一个合理的默认国旗 |
| 完全个性化 | 为每种语言自定义任意国旗、简称和全名 |
| 占用极小 | 不收集任何数据，登录时自动启动，体积仅几兆字节；唯一的网络请求是可选的每日更新检查 |

语言并不等同于国家——这正是 Flang 把国旗设计成个人选择的原因。想给法语配上
加拿大国旗，或给西班牙语配上墨西哥国旗？只需两次点击。

## 安装

从 [GitHub Releases](https://github.com/e1ernal/Flang/releases) 下载最新的
zip 压缩包，解压后把 Flang.app 移动到"应用程序"——在打开它之前请先阅读下文。

若想改为从源码构建：

```bash
git clone https://github.com/e1ernal/Flang.git
open Flang/Flang.xcodeproj
```

用 Cmd+R 构建并运行（需要 Xcode 16 或更新版本，macOS 13 Ventura 或更新版本）。

### 打开一个未签名的应用

Flang 尚未经过 Apple 公证。公证需要付费的 Apple Developer Program 会员资格，
本项目目前还没有——计划在 1.0 版本证明确实存在值得付费的用户群之后再申请（参见
[路线图](#路线图)）。在此之前，macOS 的 Gatekeeper 会阻止直接双击打开，并提示
"无法打开"Flang"，因为 Apple 无法检查其是否包含恶意软件"。

这是预期行为，不是 bug，也不代表出了什么问题。首次打开的方法：

1. 在"应用程序"中右键点击（或按住 Control 点击）Flang.app，选择**打开**。
2. 在弹出的对话框中再次点击**打开**。

只需这样操作一次——之后每次启动，包括未来的更新，都会正常打开。

## 使用方法

1. 启动 Flang——菜单栏中会出现一面国旗。
2. 点击它可切换输入源；右键点击可打开"设置"或"退出"。
3. 可选：在"系统设置 — 键盘"中取消勾选"在菜单栏中显示输入菜单"，隐藏系统自带的
   切换器。macOS 不允许应用自动完成此操作，因此这是一次性的手动步骤。

   <img src="../images/hide-system-switcher.png" width="500" alt="系统设置 — 键盘，高亮显示“在菜单栏中显示输入菜单”">

4. 要添加或移除键盘布局，打开"设置 — 输入源"，使用"+"按钮（或对某个输入源点击
   "删除"）——两者都会转到"系统设置 — 键盘 — 输入源"，由 macOS 直接处理。

   <img src="../images/add-input-source.png" width="500" alt="Flang 的输入源标签页，高亮显示“+”按钮">

## 常见问题

**Flang 会取代系统自带的切换器吗？**
在功能上是的：它使用相同的系统 API 列出并切换相同的输入源。系统自带的指示器
只能手动隐藏（见"使用方法"）。

**Flang 需要联网吗？**
不需要。唯一的可选网络请求是每天在 GitHub 上检查新版本。你或你系统的任何信息
都不会被发送到任何地方。
