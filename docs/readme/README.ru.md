# Flang

**Возвращает флаги стран в переключатель источников ввода macOS.**

[English](../../README.md) | [Español](README.es.md) | [Français](README.fr.md) | [Português](README.pt-BR.md) | [中文](README.zh-Hans.md) | [日本語](README.ja.md) | Русский

До macOS 12.4 переключатель раскладки клавиатуры в строке меню показывал флаг
страны. Затем Apple заменила флаги текстовыми обозначениями ("ABC", "RU").
Flang — небольшое приложение для строки меню, которое возвращает флаги — и
позволяет самому решать, какой флаг и какое имя представляют каждый из ваших
языков.

## Возможности

| | |
|---|---|
| Флаг в строке меню | Текущий источник ввода отображается со своим флагом, обновляется мгновенно |
| Привычное переключение | Клик открывает список всех источников ввода для переключения — как в системном меню |
| Гибкий индикатор | Флаг и имя показываются независимо друг от друга — что-то одно, оба или ничего (тогда используется системная иконка) |
| Два вида флагов | Плоские изображения флагов (классический вид) или нативные emoji-флаги |
| Разумные значения по умолчанию | Для каждой раскладки клавиатуры и метода ввода macOS подобран подходящий флаг по умолчанию |
| Полная персонализация | Любой флаг, собственное короткое и полное имя для каждого языка |
| Минимальный след | Никакого сбора данных, запуск при входе в систему, несколько мегабайт; единственный сетевой запрос — необязательная ежедневная проверка обновлений |

Языки не привязаны к странам — именно поэтому Flang делает выбор флага личным
делом. Хотите канадский флаг для французского или мексиканский для
испанского? Два клика.

## Установка

Скачайте последний DMG со страницы
[GitHub Releases](https://github.com/e1ernal/Flang/releases), откройте его и
перетащите Flang.app в Applications — перед первым запуском прочитайте раздел
ниже.

Чтобы собрать из исходников:

```bash
git clone https://github.com/e1ernal/Flang.git
open Flang/Flang.xcodeproj
```

Соберите и запустите через Cmd+R (нужен Xcode 16 или новее, macOS 13 Ventura
или новее).

### Запуск неподписанного приложения

Flang не нотаризован Apple. Нотаризация требует платного членства в Apple
Developer Program, которого у проекта пока нет — это планируется после того,
как версия 1.0 покажет, что аудитория, ради которой стоит платить, действительно
есть (см. [roadmap](#roadmap)). До тех пор Gatekeeper в macOS блокирует обычный
двойной клик с сообщением: «Flang» нельзя открыть, потому что Apple не может
проверить его на вредоносное ПО».

Это ожидаемое поведение, а не баг и не признак проблемы. Чтобы открыть в первый
раз:

1. Кликните правой кнопкой (или Control-клик) по Flang.app в Applications и выберите **Открыть**.
2. В появившемся диалоге снова нажмите **Открыть**.

## Использование

1. Запустите Flang — в строке меню появится флаг.
2. Клик переключает источник ввода; правый клик открывает Настройки и Выход.
3. По желанию: скройте встроенный системный переключатель в
   Настройках системы — Клавиатура — снимите галочку «Показывать меню ввода в
   строке меню». macOS не позволяет приложениям делать это автоматически,
   поэтому это разовое ручное действие.

   <img src="../images/hide-system-switcher.png" width="500" alt="Настройки системы — Клавиатура, с выделенным пунктом «Показывать меню ввода в строке меню»">

4. Чтобы добавить или удалить раскладку клавиатуры, откройте Настройки —
   Источники ввода и используйте кнопку «+» (или «Удалить» у источника) — оба
   действия ведут в Настройки системы — Клавиатура — Источники ввода, где
   macOS обрабатывает это напрямую.

   <img src="../images/add-input-source.png" width="500" alt="Вкладка «Источники ввода» Flang с выделенной кнопкой «+»">

## Частые вопросы

**Flang заменяет системный переключатель?**
Функционально — да: он показывает и переключает те же источники ввода через
те же системные API. Собственный индикатор системы можно скрыть только
вручную (см. раздел «Использование»).

**Нужен ли Flang интернет?**
Нет. Единственный необязательный сетевой запрос — ежедневная проверка новых
версий на GitHub. Никакие данные о вас или вашей системе никуда не
отправляются.
